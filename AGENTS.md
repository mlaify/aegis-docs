# AGENTS.md

## Project purpose
This repository contains human-facing documentation for the Aegis ecosystem.

## Rules
- Keep `aegis-spec` as the normative source for protocol behavior.
- Prefer summaries and links over duplicating full RFC text.
- Do not introduce protocol claims that are not reflected in RFC/conformance docs.
- Keep v0.1.0-alpha status language explicit and conservative.

## Build/test expectations
- Validate docs links/paths when editing cross-repo references.
- Keep release/checklist docs aligned with current implementation reality.
- If docs depend on behavior changes in other repos, call out follow-up work.

## Repo-specific notes
- `docs/` is the preferred location for active documentation.
- Top-level legacy docs may remain for compatibility, but should point toward `docs/`.
