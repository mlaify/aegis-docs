# Aegis v0.2.0-alpha Roadmap

## Overview

v0.2.0-alpha promotes the hybrid post-quantum cryptographic suite from placeholder to production. The release ships end-to-end encrypted, post-quantum-resistant messaging across the full protocol stack: core crypto, relay, gateway policy, CLI, and SDK.

Future releases will build on this foundation to deliver a complete, deployable product.

---

## v0.2.0-alpha (current)

**Theme: Production Hybrid PQ Cryptography**

### Shipped

| Area | What shipped |
|------|-------------|
| **aegis-core / aegis-crypto** | `HybridPqKeyBundle` (keygen), `HybridPqSuite` (encrypt/decrypt/sign/verify) |
| **aegis-core / aegis-proto** | `SuiteId::HybridX25519MlKem768Ed25519MlDsa65`; `EncryptedBlob` + `Envelope` PQ wire fields |
| **aegis-core / aegis-identity** | Algorithm constants; `HybridPqPrivateKeyMaterial`; self-certifying Ed25519+Dilithium3 IdentityDocument signatures; HTTP publish/resolve |
| **aegis-relay** | Accept + validate HybridPq envelopes; reject missing KEM/signature fields; SQLite WAL persistence; `PUT/GET /v1/identities/:id` endpoints |
| **aegis-gateway** | PQ-aware downgrade policy; PQ→legacy rejection by default; SMTP inbound server (RFC 5321); SMTP outbound client; IMAP4rev1 server |
| **aegit-cli** | `id init` generates full hybrid PQ key bundle; `id publish` signs + publishes IdentityDocument to relay; `msg seal` auto-detects PQ and auto-resolves recipient from relay; `msg open` dispatches on suite |
| **aegis-sdk (Rust)** | Re-exports `HybridPqSuite`, `HybridPqKeyBundle` |
| **aegis-sdk (TypeScript)** | PQ types, constants, and `SuiteId` union |
| **aegis-spec** | RFC-0005 updated; `implementation-conformance-v0.2.md` |
| **aegis-docs** | Updated security model, PQ research notes, this roadmap |

### Cryptographic Suite

`AMP-HYBRID-X25519-MLKEM768-ED25519-MLDSA65-V1`

- KEM: X25519 ECDH + ML-KEM-768 (FIPS 203) → HKDF-SHA256 → XChaCha20-Poly1305
- Signatures: Ed25519 + Dilithium3 / ML-DSA-65 (FIPS 204)
- Security: ≥ 128-bit classical, NIST Level 3 PQ; secure if either classical or PQ primitive holds

## Versioning Policy

Aegis follows `MAJOR.MINOR.PATCH[-PRERELEASE]`. During alpha, breaking wire-format changes are allowed between minor versions. The `suite_id` wire field provides backward compatibility at the envelope level regardless of library version.

See `versioning-policy.md` for details.
