# v0.1.0-alpha Milestone Checklist

Status: Working checklist aligned to current implementation.

## Scope Guardrails

- [x] Keep protocol wire shape stable for v0.1 (`Envelope`, `PrivatePayload`, identity objects).
- [x] Keep relay as zero-trust storage/transport only.
- [x] Keep demo/experimental crypto explicitly non-production.

## Implementation Reality Snapshot

- [x] Local E2E smoke test exists: `aegit-cli/scripts/local-e2e-demo.sh`
- [x] Local identity workflow exists (`id init/show/list`) with default identity pointer.
- [x] Local-dev signing key material is identity-linked and persisted alongside identity state.
- [x] Demo crypto is default for local workflows and is non-production.
- [x] `aegit msg open` reports explicit signature status:
  - unsigned
  - signature present and verified
  - signature present but failed
  - verification unavailable
- [x] `experimental-pq` exists behind feature flag and remains non-production.
- [x] Experimental `kyber768` KEM wrapper exists behind `experimental-pq`.
- [x] Gateway is policy-only groundwork (no SMTP/IMAP bridge yet).
- [x] SDK scope is thin wrapper over current v0.1 behavior.
- [x] Relay performs structural envelope validation before storage.
- [x] Relay respects `expires_at` at fetch time (expired envelopes skipped/cleaned opportunistically).
- [x] Relay supports lifecycle operations: acknowledge/delete specific envelope.
- [x] Relay supports explicit cleanup for expired envelopes and orphan ack markers.
- [x] Relay supports optional local-dev capability token gating for lifecycle-changing operations.
- [x] Relay scope remains store/fetch + health, without authn/authz or replay DB.

## Release Smoke Test (Required)

Use the local end-to-end demo as the alpha smoke test:

```sh
sh aegit-cli/scripts/local-e2e-demo.sh
```

Expected outcome:

- relay starts and answers `/healthz`
- local identity initializes
- message seals, pushes, fetches, and opens successfully
- signature verification output is shown when signature is present
- lifecycle flows are exercised (ack/fetch-empty/delete/fetch-empty/cleanup)

## Deferred Until Post-Alpha

- Production PQ deployment
- Production resolver service
- Gateway SMTP/IMAP bridge implementation
- Production signature lifecycle and trust transparency
